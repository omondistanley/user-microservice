Phase 3 Sprint Task Board (Low-Cost / Minimal Subscription)

Guardrails
- No mandatory paid SaaS in Phase 3.
- Keep architecture self-hosted: FastAPI + PostgreSQL + frontend (+ optional Redis).
- Reuse existing JWT/auth, request-id tracing, and `/api/v1` conventions.
- Prefer built-in/OSS tooling (CSV parser, Postgres jobs, in-app notifications) over subscription dependencies.

Status Legend
- TODO: not started
- IN_PROGRESS: implementation started
- DONE: merged locally and validated with tests/checks

Sprint 1 (Households + CSV Import Foundation, 1-2 weeks)

Ticket: P3-S1-T1
Title: Household Data Model (User Service)
Status: TODO
Type: DB
DB Tasks:
- Add `users_db.household`:
  `household_id (uuid pk)`, `owner_user_id`, `name`, `created_at`, `updated_at`.
- Add `users_db.household_member`:
  `household_id`, `user_id`, `role (owner|member)`, `status (active|invited|removed)`,
  `invited_by_user_id`, `created_at`, `updated_at`.
- Add constraints/indexes:
  unique (`household_id`, `user_id`), index by `user_id`, index by `status`.
Acceptance Criteria:
- Migration is idempotent.
- One user can belong to multiple households.
- Owner membership is created automatically when household is created.

Ticket: P3-S1-T2
Title: Household API (User Service)
Status: TODO
Type: API
API Tasks:
- Add endpoints:
  `POST /api/v1/households`
  `GET /api/v1/households`
  `GET /api/v1/households/{household_id}`
  `POST /api/v1/households/{household_id}/members`
  `PATCH /api/v1/households/{household_id}/members/{user_id}`
  `DELETE /api/v1/households/{household_id}/members/{user_id}`
- Add active scope endpoint:
  `PATCH /api/v1/settings/active-household` with `{ household_id | null }`.
- Enforce permissions:
  owner manages members, member has read-only household metadata.
Acceptance Criteria:
- Non-members cannot access household data.
- Member mutation endpoints reject unauthorized users with 403.
- OpenAPI includes all new household endpoints.

Ticket: P3-S1-T3

Ticket: P3-S1-T4
Title: CSV Import Pipeline (Expense Service)
Status: TODO
Type: DB/API
DB Tasks:
- Add `expenses_db.expense_import_job`:
  `job_id (uuid pk)`, `user_id`, `household_id`, `filename`,
  `status (uploaded|validated|committed|failed)`, `created_at`, `updated_at`.
- Add `expenses_db.expense_import_row`:
  `job_id`, `row_number`, `raw_payload`, `normalized_payload`,
  `validation_error`, `is_duplicate`, `created_at`.
API Tasks:
- Add endpoints:
  `POST /api/v1/expenses/import?dry_run=true|false` (multipart CSV upload),
  `GET /api/v1/expenses/import/{job_id}`,
  `POST /api/v1/expenses/import/{job_id}/commit`.
- Duplicate detection key:
  (`user_id`/`household_id`, `date`, `amount`, normalized `description`).
Acceptance Criteria:
- Dry-run does not create expenses.
- Commit only inserts valid non-duplicate rows.
- API returns counts: `total_rows`, `valid_rows`, `invalid_rows`, `duplicate_rows`, `inserted_rows`.
