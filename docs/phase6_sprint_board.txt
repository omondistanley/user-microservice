Phase 6 Sprint Task Board (UX + PWA + Accessibility)

Guardrails
- No mandatory paid SaaS in Phase 6.
- Keep frontend enhancements progressive and backward compatible.
- Maintain accessibility standards as non-negotiable acceptance criteria.
- Minimize bundle bloat; prefer existing JS/CSS architecture.

Status Legend
- TODO: not started
- IN_PROGRESS: implementation started
- DONE: merged locally and validated with tests/checks

Sprint 1 (Mobile-First UX + Offline Basics, 1-2 weeks)

Ticket: P6-S1-T1
Title: PWA Manifest + Service Worker Foundation
Status: TODO
Type: UI/Platform
Tasks:
- Add `frontend/static/manifest.json` with app icons, name, start_url, display mode.
- Register service worker from layout.
- Implement cache strategy:
  cache-first for static assets,
  network-first for API GET requests.
- Add versioned cache invalidation.
Acceptance Criteria:
- App is installable on modern mobile browsers.
- Static assets load from cache on repeat visits.
- Service worker updates cleanly on new deploys.

Ticket: P6-S1-T2
Title: Offline Queue for Expense/Income Create
Status: TODO
Type: UI/API
Tasks:
- Add local queue (IndexedDB/localStorage fallback) for create actions when offline.
- Replay queued actions on reconnect with idempotency keys.
- Add queue status UI: pending/synced/failed.
Acceptance Criteria:
- Offline-created items sync automatically when network returns.
- Duplicate creation is prevented via idempotency keys.
- Failed sync items are retriable from UI.

Ticket: P6-S1-T3
Title: Theme System (Light/Dark/System) End-to-End
Status: TODO
Type: UI
Tasks:
- Consolidate CSS vars for all major components.
- Add theme preference controls in settings/navbar.
- Apply same theme rules to chart config defaults.
- Persist preference in user settings + local fallback.
Acceptance Criteria:
- Theme applies consistently across dashboard, lists, forms, charts.
- System theme follows OS preference when selected.
- No text contrast regressions in either theme.

Ticket: P6-S1-T4
Title: Onboarding + Empty State Standardization
Status: TODO
Type: UI
Tasks:
- Add first-session onboarding hints (dismissible, once per user/device).
- Standardize empty states for dashboard, expenses, income, budgets, recurring.
- Add contextual CTAs for key actions.
Acceptance Criteria:
- New users see actionable next steps on empty pages.
- Onboarding does not reappear after dismissal unless reset.

Phase 6 Sprint 1 Definition of Done
- App is installable and resilient to intermittent connectivity.
- Theme, onboarding, and empty-state UX are consistent and polished.
- Accessibility and mobile usability meet defined acceptance standards.
- UI changes are covered by tests and operational docs.
