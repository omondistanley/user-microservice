Phase 7 Sprint Task Board (Integrations + Ecosystem)

Guardrails
- No mandatory paid SaaS in Phase 7.
- All integrations must be optional and replaceable.
- Keep provider-specific logic behind adapters/interfaces.
- Enforce idempotency and signed request validation for inbound events.

Status Legend
- TODO: not started
- IN_PROGRESS: implementation started
- DONE: merged locally and validated with tests/checks

Ticket: P7-S1-T2
Title: Webhook Ingestion Endpoint + Signature Validation
Status: TODO
Type: API/Security
Tasks:
- Add `POST /api/v1/integrations/webhooks/{provider}`.
- Validate provider signature + timestamp window.
- Ensure idempotency by storing event IDs and skipping duplicates.
- Queue webhook payload for async processing.
Acceptance Criteria:
- Invalid signatures are rejected with 401/403.
- Duplicate webhook events are no-ops with clear logs.

Status: TODO
